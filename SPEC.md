# **Webブラウザベース スクリーンキャスト編集ツール 仕様書 (Draft v3.0)**

## **1\. プロジェクト概要**

ブラウザ上で動作する ffmpeg.wasm を活用したクライアントサイド完結型の動画編集ツール（Reactベース）。

スクリーンキャスト（画面録画）の不要部分のカット、再生速度の変更、クロップ（全体/部分）、リサイズを行い、高品質な GIF または MP4 として出力することを目的とする。

*※本ツールは映像のみを扱い、音声データは破棄・無視する。*

### **技術スタック・前提条件**

* **フロントエンド:** React  
* **FFmpegライブラリ:** ffmpeg.wasm (および @ffmpeg/core) はアプリケーションにバンドルせず、**CDN (unpkg や jsdelivr など) から動的に読み込む**構成とする（初期ロード時間の短縮およびバンドルサイズの肥大化を防ぐため）。  
  * [https://ffmpegwasm.netlify.app/](https://ffmpegwasm.netlify.app/) を参照

## **2\. 主要機能要件**

### **2.1 タイムライン・編集機能（スライス管理、リファレンス実装あり）**

* **動画のインポート:** 初期画面は「Click to open or Drag & Drop」のUIとし、ローカルの動画ファイル (mp4, mov等) をブラウザに読み込む。  
* **スライスの刻み（分割）:** 再生ヘッドの位置で動画を分割し、複数の「スライス（カット）」として管理。  
* **スライスの削除:** 不要なスライスを選択して削除（Delキー対応）。  
* **スライスの伸縮（速度の自動計算）:**  
  * タイムライン上でスライスの端をドラッグして幅（再生時間）を変更可能。  
  * サイズを変更すると、元の動画の尺との比率から**再生速度（スピード倍率）が自動計算**される（例: 半分の長さに縮めると 2.0x になる）。  
* **Undo / Redo:** 操作履歴を管理し、元に戻す/やり直しが可能。

### **2.2 ビジュアル・キャンバス編集（プレビューとクロップ、リファレンス実装なし）**

* **プレビュー表示:** 現在の再生位置のフレームをキャンバスに表示。  
* **ビジュアル・クロップ（全体 / スライス個別）:**  
  * キャンバス上でバウンディングボックスをドラッグ＆リサイズし、直感的にクロップ（切り抜き）範囲を指定。  
  * **自動切り替えUI:**  
    * スライスが**未選択**の時は「全体適用」として動作する。  
    * スライスを**選択中**の時は自動的に「選択中スライス適用（個別クロップ）」モードに切り替わり、その旨がUI上に明示される。  
* **アスペクト比維持と黒塗り（パディング）対応:**  
  * クロップによって基準解像度とアスペクト比がずれた場合、映像を歪ませて引き伸ばすのではなく、アスペクト比を維持して最大化し、\*\*余白部分は黒塗り（レターボックス/ピラーボックス）\*\*にする。  
* **速度倍率のオーバーレイ表示:** 再生速度を変更したスライスにおいて、動画の右下などに速度倍率（例: x2.0, x0.5）をテキストとして自動合成するオプション。

### **2.3 出力設定（エクスポート）**

* **リサイズ指定:** 出力幅（Width）を指定。高さ（Height）はアスペクト比を維持して自動計算。  
* **出力フォーマット:** MP4 または GIF。  
* **GIF向け高品質最適化パラメータ:**  
  * FPS指定（デフォルト 10fps 等）  
  * パレット生成手法の選択（グローバルパレット or 1フレーム毎 stats\_mode=single）  
  * ディザリングアルゴリズムの選択（none, bayer, floyd\_steinberg, sierra2 等）

## **3\. FFmpeg フィルタチェーン設計 (テンプレート)**

### **黒塗り（パディング）を考慮した処理フロー**

複数のスライスを結合（concat）するためには、解像度を完全に一致させる必要があります。個別クロップでアスペクト比が変わった場合に対応するため、scale と pad フィルタを組み合わせて黒塗りを実現します。

入力動画 \-\> \[分割 & クロップ & アス比維持スケール & 黒塗りパディング & 速度変更 & テキスト\] x N \-\> \[結合\] \-\> \[リサイズ\] \-\> \[GIF最適化\] \-\> 出力

### **フィルタコマンド構築例**

* 基準解像度: 800x600（動画の元サイズ、または全体クロップのサイズ）  
* スライス1: 0秒〜5秒 (等倍, 個別クロップなし)  
* スライス2: 5秒〜10秒 (2倍速, 個別クロップ 400x150 を指定 \-\> アス比が違うため、800x600の枠内で黒塗り配置される)

**生成されるFFmpegコマンドイメージ:**

ffmpeg \-i input.mp4 \-filter\_complex "\\  
\[0:v\]trim=start=0:end=5,setpts=PTS-STARTPTS,crop=800:600:100:100\[v1\]; \\  
\[0:v\]trim=start=5:end=10,setpts=0.5\*(PTS-STARTPTS),crop=400:150:200:200,scale=800:600:force\_original\_aspect\_ratio=decrease,pad=800:600:(ow-iw)/2:(oh-ih)/2:black,drawtext=text='2.0x':x=w-tw-20:y=h-th-20:fontsize=32:fontcolor=white:box=1:boxcolor=black@0.5\[v2\]; \\  
\[v1\]\[v2\]concat=n=2:v=1:a=0\[concat\_v\]; \\  
\[concat\_v\]scale=640:-1,split \[a\]\[b\]; \\  
\[a\]palettegen\[p\]; \\  
\[b\]\[p\]paletteuse=dither=none\[out\_v\] \\  
" \-map "\[out\_v\]" \-r 10 output.gif


*解説:*

* scale=W:H:force\_original\_aspect\_ratio=decrease: 指定した枠（W:H）に収まるように、アスペクト比を維持しつつ縮小（または拡大）します。  
* pad=W:H:(ow-iw)/2:(oh-ih)/2:black: 足りない部分を黒色(black)で埋めて、最終的にW:Hの解像度に揃えます。中央配置の計算式が (ow-iw)/2 等です。

https://ffmpeg.org/ffmpeg-filters.html


## **4\. UI/UX および React コンポーネント構成案**

### **画面レイアウト構成**

1. **初期画面:**  
   * 全画面での「Click to open or Drag & drop a video file」のドロップゾーン表示。  
2. **Workspace (中央〜上部 \- 動画読込後):**  
   * **CanvasPreview (左側メイン):** \* 現在の再生フレームの表示と、オーバーレイされたクロップ領域のドラッグUI。  
     * 上部にクロップ状態のバッジ表示（例: 「🌐 全体クロップ中」「✂️ スライス個別クロップ中」等、状態に応じて自動切替）。  
   * **PropertyPanel (右側サイドバー):**  
     * **出力設定:** サイズ、FPS、GIF/MP4、ディザリング等のエクスポートオプション。  
     * **エクスポート実行ボタン**。  
3. **TimelineArea (下部):**  
   * **タイムラインコンテナ:** 上下に適度な「余白（パディング）」を持たせる。

### **状態管理 (State) のイメージ**

const state \= {  
  video: { file: File, originalDuration: 45.0, width: 3624, height: 2537 },  
  currentTime: 2.5,  
  selectedSliceId: null,  
  slices: \[  
    { id: “...”, start: 0, end: 5.0, duration: 5.0, crop: null }, // speedは original(end-start) / duration で算出  
    { id: “...”, start: 5.0, end: 10.0, duration: 2.5, crop: {x: 200, y: 200, w: 400, h: 150} }  
  \],  
  globalCrop: { x: 100, y: 100, w: 800, h: 600 },  
  exportSettings: { format: 'gif', width: 640, fps: 10, dither: 'none' }  
};  
// 速度(speed)は状態として持たず、元の尺(end \- start)と現在の表示上の長さ(duration)から派生(Derived)させて算出すると状態不整合を防げます。
